<objective>
Split each of the 4 data tiers into isolated train/val .bin files and update the dataloader API to support explicit split selection, eliminating the current calibration-evaluation leakage where PTQ and evaluation draw from the same token sets.

Purpose: Without clean train/val separation, Phase 2 measurement and Phase 5 PTQ comparison would measure on the same data used for calibration -- invalidating results.
Output: 8 new .bin files (4 train + 4 val), modified dataloader with split-aware API, and stream-splitting capability in data preparation.
</objective>
