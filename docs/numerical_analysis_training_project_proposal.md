# FP4 量化研究 — 项目方案

> 项目周期：4 周 | 最后更新：2026-05-01

---

## 一、研究目标

在 ~164M 参数的 Gemma 风格 Transformer 上，使用数值分析工具系统性研究 FP8/FP4 低精度量化的精度边界。核心问题：**FP4（仅 16 个格点）能否在实际规模的模型上实现无损推理？如果可以，需要什么技术？**

---

## 二、模型

| 参数 | 值 |
|------|-----|
| 架构 | 12 层 Transformer，RMSNorm、RoPE、GQA (4:1) |
| 隐藏维度 | 768 |

| 词汇量 | BPE 32K |
| 参数量 | ~164M |
| 训练数据 | 4.24B tokens（C4 + FineWeb + Wikipedia + OpenOrca） |

## 三、两组实验

### A 组：训练后量化（PTQ）

取已训练的 FP16 模型，无重新训练进行量化：

| 方法 | 描述 |
|--------|-------------|
| Simple（逐张量） | 对每个矩阵使用单一缩放因子进行最邻近取整 |
| Simple（逐通道） | 每个输出通道独立缩放因子 |
| GPTQ | 基于 Hessian 的列级误差补偿（Frantar et al., 2023） |
| 混合精度 | 由条件数 + Lipschitz 分析确定敏感层使用 FP8，其余使用 FP4 |

### B 组：量化感知训练（QAT）

从随机初始化开始进行 FP8/FP4 直接训练：

| 方法 | 描述 |
|--------|-------------|
| QAT-FP8 | 使用 FP8 E4M3 的 STE + 逐通道缩放 |
| QAT-FP4 | 使用 FP4 E2M1 的 STE |
| QAT + SR | 使用无偏梯度估计的随机舍入 |
| QAT + Hadamard | 用于异常值抑制的 Walsh-Hadamard 激活旋转 |
| QAT + 条件数正则化 | 对 κ(W) 添加惩罚以鼓励对量化友好的权重 |

## 四、数值分析工具

| 工具 | 方法 | 应用 |
|------|--------|-------------|
| 条件数 κ(W) | 随机幂迭代 | 逐层量化敏感度排序 |
| Lipschitz 常数 | 谱范数传播 | 跨深度的误差放大估计 |
| 敏感度报告 | κ × MSE × 传播因子 | 混合精度层分配 |
| 随机舍入 | 无偏随机量化 | 减少 QAT 中的梯度偏差 |
| 格点设计 | E2M1 / NF4 / MXFP4 | 研究数制本身的影响 |

| 指标 | 目标 |
